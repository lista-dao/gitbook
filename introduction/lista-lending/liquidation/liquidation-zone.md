# 清算区

**清算区**是Lista DAO借贷协议中的一种机制，旨在提高资产清算过程的效率和韧性。它通过启用社区驱动的清算来解决现有系统中的限制，特别是针对大规模或复杂的抵押品位置，如涉及非标准资产（例如，pt-tokens）或低流动性资产。

此功能确保在高价值清算、市场波动或网络拥堵期间系统稳定，同时激励社区参与。

### 目的

清算区补充了协议的自动清算系统，该系统依赖于荷兰式拍卖和由Lista DAO操作的白名单机器人。当官方机器人无法有效处理清算时，它会被激活，原因包括：

* **大额清算**（例如，单一位置超过100万美元）。
* **非标准资产**具有复杂结构（例如，pt-susde, pt-clisBNB），这可能在到期后迅速面临抵押率飙升。
* **清算尝试失败**经过多轮后。
* **网络限制**，如拥堵或高燃气费，中断机器人操作。

通过向社区开放清算，清算区减少了坏账风险，提高了透明度，并为参与者提供了套利机会。

### 机制

#### 核心功能

当抵押品位置满足预定义条件时，它将被转移到**清算区**，允许社区成员参与清算过程。社区清算者可以通过链上交易以折扣价格获取资产，在Lista DAO收取固定罚款费用后保留利润。过程如下：

1. **触发检测**：机器人根据预定义条件识别符合清算区的位置。
2. **团队通知**：机器人向Lista DAO团队报告位置详情，包括：
   * 清算ID
   * 平台（CDP或借贷）
   * 市场和资产（例如，BTCB/USD1用于抵押/债务）
   * 抵押和债务金额（资产和美元等值）
3. **手动激活**：Lista DAO团队手动启用该位置的社区清算。
4. **社区清算**：符合条件的社区成员执行清算，利用套利机会。
5. **权限撤销**：清算后，团队撤销社区访问权限以维持协议控制。

#### 示例机器人通知

```
公开清算警报
- 清算ID：xxx
- 平台：CDP/借贷
- 市场/资产：BTCB/USD1（抵押/债务）
- 抵押金额：xxx BTCB（约xxx USD）
- 债务金额：xxx USD1（约xxx USD）
```

### 进入清算区的条件

当满足以下条件之一时，抵押品位置有资格进入清算区：

1. **清算价值阈值**：
   * 该位置的清算价值超过100万美元。
2. **重复失败的清算**：
   * 官方机器人在多次尝试后（例如，超过1小时）未能清算位置，触发机器人警报。
3. **手动指定**：
   * 协议操作员手动标记位置进入清算区，通常是为了应对特殊情况（例如，安全漏洞或紧急情况）。

### 关键特性

* **社区参与**：使具有套利能力的社区成员能够清算资产，提高效率。
* **利润激励**：清算者以折扣价格获取资产，在支付固定罚款费用后保留利润。
* **系统韧性**：减轻大规模清算、非标准资产或网络中断的风险。
* **透明度**：公开警报和开放过程确保可见性和信任。

### 与现有系统的整合

清算区与Lista DAO现有的清算框架并行运作，包括：

* **CDP清算**：
  * 白名单机器人（Lista DAO操作）。
  * 从抵押品估值的110%开始的荷兰式拍卖（10%罚款）。
  * 每个区块的价格降低，直到达到最大降价次数。
* **借贷清算**：
  * 白名单机器人。
  * 从债务的106%开始的荷兰式拍卖（6%罚款，可调）。
  * 主要位置的闪电清算和残余金额的尾部清算。

仅当这些机制不足时，清算区才会激活，确保向社区驱动过程的无缝过渡。

### 安全考虑

* **受控访问**：仅针对特定位置启用社区清算，并在清算后撤销，以防止未经授权的访问。
* **机器人监控**：持续的机器人警报确保及时检测符合条件的位置。
* **手动保障**：团队监督在激活期间减轻风险并确保协议完整性。

### 未来增强

* **自动触发器**：探索基于燃气价格或特定资产的自动化，以减少手动干预。
* **扩展资产支持**：增强与额外非标准或低流动性资产的兼容性。
* **社区工具**：开发界面或指南，简化社区参与清算区的过程。
